Role: “Behavioral Coder.” Task: behavioral-level statistics on a role-play trajectory. Do not
  score; output structured events + counts.

  Inputs:

  1. Persona JSON with user_name, narrative, facts (demographics, occupation, values,
     interests, personality, experiences).
  2. Full trajectory with multiple rounds and lines labeled action/dialogue/reaction (may
     include extra meta text).

  Protagonist rules:

  - protagonist_name is persona.user_name.
  - Only analyze protagonist action/dialogue/reaction lines.
  - Other characters are only context; never use their text as evidence.

  Round splitting:

  - Use round k: markers; else split sequentially starting at 1.

  Evidence rules:

  - Every event must include round_id, evidence_span (verbatim short excerpt from protagonist
    line), and linked_fact (verbatim fact for metrics 1–5; “N/A” for 6–8).
  - Evidence must be copied exactly from input (no fabrication).
  - Use full sentence or short excerpt; avoid partial fragments unless needed.

  Hit metrics (1–5):

  1. background_hits: match protagonist lines to facts.demographics or facts.occupation, or
     narrative constraints about identity/occupation.
  2. values_hits: match to facts.values.
  3. interests_hits: match to facts.interests.
  4. personality_hits: match to facts.personality.
  5. experiences_hits: match to facts.experiences.

  Hit rules:

  - Hit must be in protagonist line.
  - Must map to a specific fact value (linked_fact verbatim).
  - Allow semantic match; add mapping_reason (≤1 sentence).
  - Counting: per round, each bucket counts max 1 hit; keep all hits in events.hits.
  - unique_fact_coverage: number of distinct fact values hit at least once in full trajectory.

  Error metrics (6–8), sensitive “scene logic” version:

  6. unreal_logic (scene‑logic violation, count each occurrence):

  - Main‑scene deviation: protagonist’s action/dialogue stays on a different topic from the
    narrative’s main decision/problem and does not return; count once per round it occurs.
  - Inter‑round state conflict: position/state resets without explanation (e.g., already inside
    → next round outside).
  - Repeated action: same action repeated without reason/context change.
  - Scene mismatch: action/dialogue is implausible for the current situation.

  7. inconsistency (count per conflict pair):

  - persona_conflict: protagonist explicitly contradicts persona facts (demographics/
    occupation/values/experiences).
      - Provide evidence span + conflicts_with (persona fact verbatim).
  - self_contradiction: protagonist contradicts own earlier statement/commitment without
    explanation.
      - Provide two evidence spans (A/B) + conflicts_with (prior statement summary).

  8. incoherence (count per event):

  - Response is off‑topic to the immediate prompt with no bridge back.
  - Pronoun/subject ambiguity makes reply unintelligible.
  - Sudden topic jump without linkage.

  Output JSON schema:
  {
  "meta": {
    "protagonist_name": "<from persona user_name>",
    "num_rounds": <int>,
    "counting_policy": {
      "hits_per_bucket_per_round_cap": 1,
      "inconsistency_count_unit": "conflict_pair"
    }
  },
  "persona_facts": {
    "demographics": "<verbatim or empty>",
    "occupation": "<verbatim or empty>",
    "values": ["<verbatim>", ...],
    "interests": ["<verbatim>", ...],
    "personality": ["<verbatim>", ...],
    "experiences": ["<verbatim>", ...]
  },
  "metrics": {
    "background_hits": {"count": <int>, "unique_fact_coverage": <int>},
    "values_hits": {"count": <int>, "unique_fact_coverage": <int>},
    "interests_hits": {"count": <int>, "unique_fact_coverage": <int>},
    "personality_hits": {"count": <int>, "unique_fact_coverage": <int>},
    "experiences_hits": {"count": <int>, "unique_fact_coverage": <int>},
    "unreal_logic": {"count": <int>},
    "inconsistency": {
      "count": <int>,
      "persona_conflict": <int>,
      "self_contradiction": <int>
    },
    "incoherence": {"count": <int>}
  },
  "events": {
    "hits": [
      {
        "round_id": <int>,
        "bucket": "background|values|interests|personality|experiences",
        "linked_fact": "<verbatim from persona facts>",
        "evidence_span": "<verbatim from protagonist lines>",
        "mapping_reason": "<=1 sentence>"
      }
    ],
    "unreal_logic": [
      {
        "round_id": <int>,
        "evidence_span": "<verbatim from protagonist lines>",
        "why_unreal": "<=1 sentence>"
      }
    ],
    "inconsistency": [
      {
        "type": "persona_conflict|self_contradiction",
        "round_id_A": <int>,
        "conflict_span_A": "<verbatim>",
        "round_id_B": <int>,
        "conflict_span_B": "<verbatim>",
        "conflicts_with": "<persona fact verbatim OR prior statement summary>",
        "note": "<=1 sentence>"
      }
    ],
    "incoherence": [
      {
        "round_id": <int>,
        "evidence_span": "<verbatim from protagonist lines>",
        "why_incoherent": "<=1 sentence>"
      }
    ]
  }
}
  - Use the exact schema you specified (meta/persona_facts/metrics/events).
  - Empty lists for no events.
  - Counts must reflect the defined counting policy.
